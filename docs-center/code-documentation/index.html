Engine-first architecture

Code Documentation

React renders state and submits actions; game rules live in pure TypeScript modules under src/engine/.

Module Map

ModuleResponsibilityReuse Notes
src/engine/math.tsPrime factorization, multisets, Harmony, shared primes, dice pool counts, integer log2.Adapted from Primordial Cipher prime math concepts.
src/engine/dice.tsd6 rollers, seeded/sequence test rollers, combat die scoring.Uses Primordial Cipher's dependency-injected dice pattern.
src/engine/game.tsAction dispatcher, turn flow, captures, optional actions, combat choices, end turn.Primary contract for UI and future AI or multiplayer commands.
src/engine/combat.tsAttack eligibility, Prime Theft availability, combat roll resolution.Preserves repeated prime multiplicities.
src/engine/race.tsSprint, Run, and Marathon Harmony targets plus Foothill, Mountain, and Everest distinct-prime targets.Serializable race configuration.
src/engine/resonance.tsOwned-prime threshold and excess-halving rule.Isolated so playtest balance can change safely.
src/engine/turbulence.tsRepeated-prime accounting, optional-action surcharge, and end-turn smoothing.Pure rule helpers shared by engine, UI, and AI.
src/ui/soundEffects.tsAction-to-sound routing, audio caching, and sound-effects mute state.Kept in the UI so engine rules remain deterministic.
src/ui/RaceTracker.tsxResponsive two-axis race visualization and plot scaling.Reads serializable engine state without owning game rules.
src/ai/heuristic.tsDecision ownership and explainable heuristic action selection.UI-independent policy over the same engine action contract used by humans.
src/ui/musicLibrary.tsBuild-time discovery and presentation metadata for jukebox tracks.New audio files require no manual jukebox registration.
src/ui/devMatchLogger.tsDevelopment-only action snapshots, local archive, and JSON export.Feeds playtest logs without entering authoritative engine state.
src/shared/account/Federated server adapters, session contracts, and session codec.Game-neutral extraction target for Primordial Cipher.
src/shared/protocol/Canonical serialization, SHA-256 adapter, command/event envelopes, and validation results.Domain-separated and independent from Collatz rules.
src/Portal.tsxGateway, Hub, Race, and Account scene orchestration.Networking remains behind account adapters.
src/App.tsxAccount cockpit, selected-rival query, focused action board, and paced local AI orchestration.Should stay rule-thin.

Next Milestone: Deterministic Replay

Canonical serialization and hashing are foundations, not yet a replay system. The next refactor replaces function-valued random rollers with explicit protocol inputs, validates sequenced commands, emits typed events, hashes resulting states, and proves that a full transcript reconstructs the same final state from genesis.

Future Team Modes

MVP ownership is per-player. Future variants should split pulse ownership, resonance ownership, captured-factor ownership, and victory ownership before shared team pools are added.
